# 4주차 실습 정리

## 오늘 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰 및 AI 활용하여 직접 프로그래밍

## 관찰 내용

### 원형 vs AABB
원형은 캐릭터 중심을 기준으로 반지름만큼 충돌을 계산해서 부드럽게 겹침 감지가 된다. 또한 AABB는 항상 축에 평행한 사각형 기준이라, 회전된 오브젝트와 충돌할 때 실제 모양과 차이가 있다. 따라서 원형은 회전이나 돌, 폭팔 같은 오브젝트에서 더 좋다.

### AABB vs OBB
OBB는 오브젝트 회전까지 고려해서 충돌 판정이 정확하게 따라간다. AABB는 회전 시 충돌 판정이 커지거나 실제 모양과 차이가 나서 정황석이 떨어진다. 따라서 회전이 중요한 경우 OBB가 더 나은 충돌 판정을 가진다.

## AI와의 대화에서 배운 것
충돌 판정 방식별 장단점과 구현 방법을 단계별로 쉽게 이해할 수 있었으며, 삼각형, 직사각형 등 다양한 도형에도 적용할 수 있도록 SAT(Separating Axis Theorem)을 활용하는 방법을 배웠고, AI를 활용하면 코드 구조와 회전/충돌 로직을 동시에 고민하게 되었다.

## 내 게임에 적용한다면
포아너 처럼 무기와 무기가 서로 만날 때 이 시스템을 적용시키고 시스템에 따라 패링, 피격판정, 가드성공등을 적용시키고 싶다.
